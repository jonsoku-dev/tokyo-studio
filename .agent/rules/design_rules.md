# 디자인 규칙 (Design Rules)

## 핵심 철학: 심플함 & 직관성 (Simple & Intuitive)
"사용자가 생각하게 하지 마라 (Don't Make Me Think)"
복잡한 취업/이민 프로세스를 다루지만, 인터페이스만큼은 Reddit처럼 익숙하고 편안해야 합니다.

### 1. 인지 부하 감소 (Cognitive Load Reduction)
*   **정보의 계층화**: 모든 화면은 '가장 중요한 정보' 하나만 강조합니다. 부가 정보(작성일, 조회수 등)는 시각적으로 후순위처리(음영, 작은 폰트)합니다.
*   **익숙한 패턴 사용**: 새로운 학습이 필요한 독창적인 UI보다는, Reddit/GitHub 등 개발자에게 익숙한 UX 패턴을 차용합니다.
*   **예측 가능한 동작**: 버튼은 기대하는 대로 동작해야 하며, 클릭 결과가 명확해야 합니다.

## 심미성 & UX (Aesthetics & UX)
*   **프리미엄 미니멀리즘**: 장식적인 요소는 배제하고, 여백(Whitespace)과 타이포그래피(Inter, Pretendard)로 고급스러움을 연출합니다.
*   **마이크로 인터랙션**: 버튼 Hover, 모달 등장, 탭 전환 시 부드러운 애니메이션(Transition)을 적용하여 '살아있는' 느낌을 줍니다.
*   **명확한 피드백**: 저장 성공, 로딩 중, 에러 발생 시 즉각적이고 명확한 피드백을 제공합니다 (Toast, Skeleton UI).

## 기술 및 컴포넌트 표준 (Tech & Components)
*   **CSS 프레임워크**: TailwindCSS v4
*   **인터랙션**: Headless UI / Radix UI (접근성 및 키보드 네비게이션 보장)
*   **아이콘**: LucideReact (일관된 라인 스타일)
*   **폰트**: Inter (영문/숫자), Pretendard (국문)

## 디자인 레퍼런스 적용 전략 (Design Application Strategy)

### 1. Reddit 스타일 피드 (The Feed Pattern)
**핵심**: 밀도(Density)와 스캐닝(Scanning)
*   **카드형 컨테이너**: 각 정보(할 일, 공고)는 배경과 구분된 독립된 카드로 디자인합니다.
*   **좌측 액션 바**: Reddit의 Upvote처럼, '체크박스'나 '상태 뱃지'를 카드 좌측에 배치하여 즉각적인 상태 파악과 액션을 유도합니다.
*   **적용 대상**: 대시보드(오늘의 할 일), 채용 공고 리스트(Pipeline).

### 2. 계층형 상세 뷰 (Threaded Detail View)
**핵심**: 몰입(Immersion)과 계층(Hierarchy)
*   **콘텐츠 집중 모달**: 상세 내용은 오버레이 모달로 띄워 주변 방해 없이 콘텐츠에 집중하게 합니다.
*   **계층형 댓글(Threaded UI)**: 멘토링 피드백이나 면접 후기 댓글은 '연결선(Vertical Line)'을 사용해 대화의 깊이(Depth)를 시각화합니다.
*   **적용 대상**: 멘토링 리포트 상세, 문서 리뷰 코멘트, 면접 후기 Q&A.

### 3. GitHub 스타일 대시보드 (Activity Dashboard)
**핵심**: 시각화(Visualization)와 성취감
*   **잔디심기(Contribution Graph)**: 12주 로드맵의 진행 상황을 히트맵이나 그래프로 시각화하여 꾸준한 활동을 독려합니다.
*   **Clean List**: 다량의 데이터(내 문서함 등)는 파일 탐색기처럼 장식 없는 리스트 뷰로 라인 간격을 좁혀 효율적으로 보여줍니다.
*   **적용 대상**: 로드맵 진행률, 내 문서함, 활동 내역.

### 4. 직관적인 입력 폼 (Intuitive Submission)
**핵심**: 단순화(Simplification)
*   **탭/단계별 입력**: 복잡한 입력(진단, 이력서)은 한 페이지에 쏟아붓지 않고, 탭이나 단계(Wizard)로 쪼개어 인지 부하를 줄입니다.
*   **컨텍스트 도움말**: 입력 폼 우측이나 옆에 '작성 가이드' 패널을 고정하여, 사용자가 입력 도중 이탈하지 않고 도움을 받게 합니다.
*   **적용 대상**: 온보딩 진단, 이력서 작성, 커뮤니티 글쓰기.

## 도메인별 매핑 요약 (Domain Mapping Summary)

| 기능 (Feature) | UI 패턴 (UI Pattern) | 디자인 의도 (Design Intent) |
| :--- | :--- | :--- |
| **로드맵 (Roadmap)** | **GitHub Dashboard + Activity** | 12주 여정을 프로젝트 관리하듯 시각화. 완료 시 잔디가 채워지는 성취감 제공. |
| **파이프라인 (Pipeline)** | **Feed Cards** | 각 지원 건을 피드 카드처럼 나열. 좌측 뱃지로 상태(서류/면접/합격) 직관적 인지. |

| **문서 리뷰 (Review)** | **Threaded Comments** | 코멘트와 답변을 연결선으로 이어주어, 문서의 어느 부분에 대한 논의인지 명확화. |
