# 디자인 레퍼런스 분석 보고서 (Design Reference Analysis)

본 문서는 `designs/` 디렉토리에 포함된 레퍼런스 이미지(Reddit, GitHub)를 심층 분석하여, **일본 IT 취업 플랫폼**에 적용할 구체적인 UX/UI 패턴을 도출합니다.

## 1. 개요 (Overview)
우리의 목표는 **"심플하고 직관적인"** 경험입니다. 이를 위해 사용자가 이미 학습한 **Reddit(커뮤니티/피드)**과 **GitHub(대시보드/관리)**의 디자인 문법을 차용합니다.

| 레퍼런스 파일 | 분석 대상 | 핵심 키워드 | 적용 기능 |
| :--- | :--- | :--- | :--- |
| `www.reddit.com_.png` | 피드 & 리스트 | 밀도(Density), 스캐닝(Scanning) | 대시보드, 할 일 목록, 채용 공고 |
| `reddit_post.png` | 상세 & 토론 | 몰입(Immersion), 계층(Hierarchy) | 멘토링 리포트, 면접 후기, 문서 리뷰 |
| `www.reddit.com:submit.png` | 입력 폼 | 단순화(Simplification) | 진단 실행, 이력서 작성 |
| `github.png` | 활동 내역 | 시각화(Visualization) | 로드맵 진행률, 기여도 그래프 |

---

## 2. 상세 분석 및 적용 전략

### 2.1. Reddit 스타일 피드 (The Feed Pattern)
**레퍼런스**: `www.reddit.com_.png`

**분석 (Analysis):**
*   **카드형 컨테이너**: 각 정보(게시글)가 독립된 카드 안에 담겨 있어 정보의 경계가 명확합니다. 배경색(Gray-100)과 카드색(White)의 대비를 통해 입체감을 줍니다.
*   **좌측 액션 바 (Vote/Status)**: Reddit의 Upvote/Downvote 영역은 사용자가 콘텐츠에 대해 즉각적인 액션을 취할 수 있는 공간입니다.
*   **메타데이터의 위치**: 작성자, 시간, 태그 등 부가 정보는 카드 상단이나 하단에 작고 흐리게 배치하여 제목의 가독성을 해치지 않습니다.

**플랫폼 적용 (Application):**
*   **[대시보드 > 오늘의 할 일]**:
    *   각 '할 일'을 카드로 디자인합니다.
    *   Reddit의 Upvote 위치에 **'체크박스'**를 배치하여 완료 시 즉각적인 쾌감을 줍니다.
    *   카드 클릭 시 상세 모달을 띄웁니다.
*   **[채용 공고 (Pipeline)]**:
    *   각 공고를 카드로 나열합니다.
    *   Upvote 대신 **'지원 상태(Applied, Interview)'** 뱃지를 좌측에 배치하여 현재 상태를 한눈에 파악하게 합니다.

### 2.2. 계층형 상세 뷰 (Threaded Detail View)
**레퍼런스**: `reddit_post.png`

**분석 (Analysis):**
*   **콘텐츠 집중 모달**: 배경을 어둡게 처리(Overlay)하고 중앙에 콘텐츠 팝업을 띄워 사용자의 시선을 집중시킵니다.
*   **Threaded Comments (대댓글)**: 댓글과 대댓글을 '연결선(Vertical Line)'으로 시각화하여 대화의 깊이(Depth)를 표현합니다. 이는 복잡한 논의를 구조적으로 보여주는 데 탁월합니다.

**플랫폼 적용 (Application):**
*   **[문서 리뷰 (Document Review)]**:
    *   멘토의 피드백을 Reddit 댓글처럼 시각화합니다.
    *   특정 문단에 대한 코멘트를 '대댓글' 형태로 달아 문맥을 유지합니다.
*   **[면접 후기 상세]**:
    *   면접 질문과 답변을 Q&A 형식으로 보여주고, 그 밑에 달린 사용자들의 조언이나 추가 질문을 계층형으로 보여줍니다.

### 2.3. GitHub 스타일 대시보드 (Activity Dashboard)
**레퍼런스**: `github.png`, `github.com_.png`

**분석 (Analysis):**
*   **사이드바 네비게이션**: 좌측에 Repository(프로젝트) 목록을 고정하여 언제든 컨텍스트를 전환할 수 있습니다.
*   **잔디심기 (Contribution Graph)**: 사용자의 활동량을 시각적인 히트맵으로 보여주어 성취감을 고취시킵니다.
*   **Clean List**: 파일 탐색기처럼 군더더기 없는 리스트 뷰는 많은 양의 데이터를 다룰 때 효율적입니다.

**플랫폼 적용 (Application):**
*   **[로드맵 (Roadmap)]**:
    *   12주간의 여정을 GitHub 잔디심기 그래프처럼 시각화하여 **'꾸준한 활동'**을 독려합니다.
    *   주차별 과제 리스트는 GitHub 파일 탐색기처럼 간결한 라인 형태로 디자인합니다.
*   **[내 문서함 (My Documents)]**:
    *   작성 중인 이력서 버전을 리스트 형태로 관리합니다.

---

### 2.4. 직관적인 입력 폼 (Intuitive Submission)
**레퍼런스**: `www.reddit.com:submit.png`

**분석 (Analysis):**
*   **탭 기반 유형 선택**: Text, Image, Link 등 게시글 유형을 상단 탭으로 먼저 선택하게 하여 입력 폼을 단순화합니다.
*   **우측 가이드 패널**: 입력 폼 우측에 '작성 규칙(Rules)'을 고정하여, 사용자가 입력 중에 실시간으로 도움말을 참조할 수 있습니다.

**플랫폼 적용 (Application):**
*   **[진단 (Diagnosis) 실행]**:
    *   복잡한 설문을 한 페이지에 다 넣지 않고, 탭이나 단계(Step)로 쪼갭니다.
    *   우측에 **'왜 이 정보가 필요한가요?'** 패널을 두어 진단의 신뢰도를 높입니다.
*   **[커뮤니티 글쓰기]**:
    *   면접 후기 작성 시, Reddit처럼 유형(일반/후기/질문)을 먼저 선택하게 하고 그에 맞는 템플릿을 제공합니다.

## 3. 결론 (Conclusion)
우리의 디자인은 **"Reddit의 구조적 명확함"**과 **"GitHub의 성취감 시각화"**를 결합합니다.
- **Feed**: 계속해서 새로운 기회(공고, 멘토링)를 탐색하는 경험.
- **Thread**: 깊이 있는 피드백과 지식을 축적하는 경험.
- **Activity**: 자신의 성장을 눈으로 확인하는 경험.

이 세 가지 기둥을 중심으로 디자인 시스템을 구축합니다.
