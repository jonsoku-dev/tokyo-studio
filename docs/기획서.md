일본 IT 취업 전 과정 지원 플랫폼 기획서 (v1.0)

작성일: 2025-12-27 (JST)

대상: 한국어 사용자(한국 거주/일본 거주 포함) + 일본 취업을 목표로 하는 IT 전 직군

1. 한 줄 정의
    
    한국인이 “일본 IT 취업 → 비자/입국 → 초기 정착 → 커리어 성장”까지 끊기지 않도록, 개인별 전략·산출물·지원 파이프라인·멘토링·커뮤니티를 하나의 상태기계로 운영하는 웹 플랫폼.
    
2. 배경과 문제 정의(리서치 근거 포함)
    
    2.1 왜 지금 필요한가
    

  

- 일본은 외국인 노동자 규모가 증가했고, 2024년 10월 말 기준 외국인 노동자는 2,302,587명(전년 대비 +253,912명, +12.4%)으로 “전문적·기술적 분야” 체류자격이 가장 큰 비중으로 올라왔다는 정부 발표가 있다. 즉 “전문직 외국인 채용”은 구조적 흐름이다. (후생노동성) 
    
- 비자 제도도 “우수 인재” 유치를 강화했다. 예: J-Skip(특별고도인재) 도입(출입국재류관리청)  , J-Find(미래창조인재) 제도 안내 PDF(법무성/출입국) 
    
- “영어 중심” 채용 채널은 실재하고, 해외에서 지원 가능한 포지션을 전면에 내세운 잡보드도 존재한다(TokyoDev, Japan Dev). 
    
- “일본어 중심” 채용 채널은 규모가 크고 스카우트형이 강하다(예: Green의 스카우트 활용 가이드, Forkwell Jobs의 엔지니어 특화 설명). 
    

  

2.2 사용자가 실제로 겪는 실패 지점(플랫폼이 해결해야 하는 것)

- 정보 파편화: 영어 채널/일본어 채널/에이전트/공공지원이 분리되어 전략을 못 정함
    
- “학습”은 하지만 채용 가능한 증거(프로젝트·성과·문서·커뮤니케이션)로 못 묶음
    
- 지원·면접·오퍼·비자·입국의 일정/문서 누락으로 기회가 새거나 리드타임이 붕괴
    
- 입국 후 14일 내 거주지 등록 같은 필수 절차를 놓쳐 온보딩이 지연됨(정부 체크리스트·절차 문서에 14일 규정이 명시). 
    
- 커리어 유지/이직 단계에서 신뢰할 네트워크(레퍼럴, 내부 정보, 오퍼 비교)가 없음
    

  

3. 비전, 목표, 비목표
    
    3.1 비전
    
    “일본 IT 취업을 ‘운’이 아니라 ‘운영’으로 만든다.”
    

  

3.2 제품 목표(성과 지표 포함)

- 오퍼/입사 성과를 내는 것(최우선)
    
- 준비 과정의 낭비를 줄이는 것(산출물 중심)
    
- 입사 후 90일 유지(정착) 비율을 올리는 것
    

  

핵심 KPI

- Activation: 온보딩 진단 완료율, 로드맵 생성율
    
- Output: 지원 파이프라인 생성 수, 서류 제출 수, 면접 진행 수, 오퍼 수(증빙 선택)
    
- Quality: 멘토 평점/재구매율, 커뮤니티 신고율 대비 해결 SLA, 후기 구조화 필드 완성도
    
- Retention: 4주·12주·24주 리텐션, 입사 후 90일 체크인 완료율
    

  

3.3 비목표(초기에 안 한다)

- 공고 스크래핑/무단 재배포(약관·저작권·차단 리스크)
    
- 유료 알선/수수료형 소개(법무·컴플라이언스 설계 없이 시작하면 폭발)
    
- “모든 도시의 정착 행정 완벽 커버”(도쿄 템플릿 → 핵심 도시 확장)
    

  

4. 타겟 사용자 세그먼트와 니즈
    
    A. 신입(대학생/졸업생/부트캠프/독학)
    

  

- 니즈: 직무 선택, 포트폴리오 품질, 일본식 서류·면접 구조, 채널 선택(영어/일본어)
    
    B. 일본 유학생(신졸/주니어)
    
- 니즈: 체류자격 변경/취업 전략, 일본어 비즈니스 커뮤니케이션, 현지 네트워크
    
    C. 커리어 전환(비전공 포함)
    
- 니즈: 전환 서사, 최소 증거 세트, 시간·리스크 관리(최단 루트)
    
    D. 경력직(미들~시니어)
    
- 니즈: 채널 믹스(스카우트/에이전트/영어권), 오퍼 비교, 빠른 매칭, 장기 체류 전략(고도인재/영주)
    
    - 고도인재 포인트 기반 영주 신청 완화는 출입국 문서에서 80점(1년), 70점(3년) 조건을 명시한다. 
        
    

  

5. 서비스 컨셉: 5단계 “상태기계” 설계
    
    핵심은 “콘텐츠 사이트”가 아니라 “사용자 상태 + 다음 행동”을 강제하는 운영체제다.
    

  

상태(예시)

- Discover(탐색) → Decide(경로결정) → Build(산출물) → Apply(지원) → Interview → Offer → Visa/COE → Arrive → Onboard(30/60/90) → Grow → Switch(이직)
    

  

각 상태는:

- 필수 산출물(문서/링크/체크리스트)
    
- 필수 행동(지원/첨삭/모의면접/행정)
    
- 실패 리스크(누락 시 결과)
    
    를 가진다.
    

  

6. 기능 범위(모듈)와 MVP 우선순위
    
    6.1 MVP(1차 출시에서 “반드시”)
    
    (1) 온보딩 진단 + 개인 로드맵 생성
    

  

- 입력: 직군/레벨/경력, 일본어/영어, 지역(도쿄 우선), 학력/비자 제약, 희망 근무형태(원격/출근)
    
- 출력:
    
    - 채널 전략(영어권/일본어권/스카우트/에이전트) 조합
        
    - 12주 플랜(학습이 아니라 “산출물” 중심)
        
    - 지원 파이프라인 초기 템플릿
        
    - 추천 멘토 패키지(진단/서류/면접 중 최소 1개)
        
    

  

(2) 지원 파이프라인 트래커(칸반 + 리마인더)

- 공고는 “링크 저장 + 노트 + 상태”로 관리(스크래핑 불필요)
    
- 상태: 관심→지원→과제→면접(1/2/3)→오퍼→비자/COE→입사
    
- 일정: 면접/과제 마감/서류 업데이트 주기
    
- 문서 버전 연결: 이력서/職務経歴書/영문 CV/포트폴리오
    

  

(3) 문서/템플릿 센터 + 리뷰 워크플로우

- 템플릿: 이력서(일/영), 職務経歴書, 자기PR/志望動機, 면접 답변(STAR), 오퍼 비교표
    
- 리뷰 흐름: “요청 → 멘토 배정 → 코멘트 → 리비전 → 완료”
    
- 버전 관리: 제출했던 버전 재현 가능(지원 성과 분석에 필요)
    

  

(4) 멘토링 예약(초기엔 큐레이션 모델)

- 멘토 유형: 진단(전략), 서류, 면접
    
- 품질장치: 승인제 + 세션 후 평점/리포트 제출 강제
    

  

(5) 커뮤니티(구조화된 데이터 수집형)

- 게시판 최소 구성:
    
    - 경로/채널별 Q&A(영어권/일본어권/에이전트)
        
    - 면접 후기(구조화 필드 강제)
        
    - 포트폴리오 피드백(템플릿 강제)
        
    
- “자유글만” 허용하면 데이터가 썩는다. 처음부터 구조화가 핵심.
    

  

(6) 도쿄 정착 체크리스트(최소)

- 정부 가이드/절차 문서가 명시하는 “14일 내 거주지 등록” 같은 핵심만이라도 타임라인으로 제공. 
    

  

6.2 2차(성장 기능)

- 스터디/프로젝트 팀 매칭(협업 증거 강화)
    
- 포트폴리오 빌더(프로젝트 구조화 + 링크/배포/ADR)
    
- 지역 확장(오사카/후쿠오카 등) 정착 템플릿
    
- 경력직 모드(오퍼 비교 고도화, 연봉·조건 협상 코칭)
    

  

6.3 3차(파트너십)

- 기업 제휴(비자 스폰서 경험 기업) → 플랫폼 내 원클릭 지원
    
- 레퍼럴 보드(검증 회원만 글/연결 가능)
    
- 공공 지원 연계(선택): KOTRA K-Move/WorldJob/HRDK 정착지원금 안내(사용자에게 “옵션”으로 제공)
    
    - KOTRA K-Move 사업 소개 
        
    - HRDK 해외취업정착지원금(최대 500만원, 3차 분할 등) 
        
    

  

7. 채널 전략(영어 기반 vs 일본어 기반) 제품화
    
    7.1 제품 내 “채널 카테고리”
    

  

- 영어권 잡보드/커뮤니티: TokyoDev(영어권 소프트웨어 엔지니어 잡보드+커뮤니티), Japan Dev(해외 지원 가능 포지션 다수, 일본어 유창 요구가 적다고 명시) 
    
- 일본어권 스카우트/전직: Green(스카우트 활용 비중을 강조), Forkwell Jobs(엔지니어 특화, 스카우트 강조) 
    
- 공공 취업지원(도쿄): 도쿄 외국인 고용서비스센터(신주쿠 권역 2개 센터) 
    

  

7.2 채널 선택 로직(진단 결과로 자동 추천)

- 일본어 낮음 + 경력 있음: 영어권 채널 우선 + “일본어 성장 플랜” 병행
    
- 일본어 중상 + 신입/주니어: 일본어권 채널 비중 확대 + 서류/면접 일본식 최적화
    
- 경력직: 스카우트/에이전트/하이클래스 조합(단, MVP에선 링크·전략 제공 중심)
    

  

8. 사용자 여정(대표 플로우 4개)
    
    8.1 신입(부트캠프) 플로우
    

  

- 진단 → 12주 산출물 플랜(프로젝트 2개 + 테스트/문서 포함) → 포트폴리오 템플릿 적용 → 서류 템플릿 생성 → 멘토 서류리뷰 → 지원 트래커로 주간 목표(주 10건) 운영 → 면접 후기 구조화 기록 → 오퍼 비교표
    

  

8.2 커리어 전환 플로우

- 진단(이전 경력 강점 추출) → “전환 서사 템플릿”으로 경력서 구성 → 최소 증거 세트(실무형 토이 프로젝트 + 문서) → 모의면접(전환 질문 집중) → 지원/면접 운영
    

  

8.3 경력직 플로우

- 진단(언어·연봉·지역) → 채널 믹스 제안(영어권 + 스카우트) → 경력서 성과 중심 리라이트 → 오퍼 비교/협상 코칭 → 장기 체류 옵션(고도인재/영주) 정보 제공
    
    - HSP 포인트 기반 영주 신청 완화(80점 1년, 70점 3년 조건을 출입국 문서가 명시) 
        
    

  

8.4 일본 도착 후 플로우(정착)

- 입국일 입력 → D+14 체크리스트 자동 생성(거주지 등록 등) → 30/60/90일 온보딩 체크(회사 적응/성과) → 커뮤니티 지역 채널 참여

11. 권한/운영 모델(망가지기 쉬운 부분을 먼저 고정)
    
    11.1 멘토 품질 통제(필수)
    

  

- 승인제(경력/재직 증빙은 “선택 + 블라인드 검수”로 설계)
    
- 세션 종료 시 “결과 리포트” 제출 강제(사용자 로드맵에 자동 반영)
    
- 평점/재구매율이 낮으면 노출 제한
    

  

11.2 커뮤니티 신뢰 모델(필수)

- 면접 후기/오퍼 정보는 구조화 필드 강제 + 민감정보 차단
    
- 신고/차단/검수 SLA 정의(예: 24시간 내 1차 처리)
    
- “레퍼럴”은 검증 회원만(초기엔 기능 자체를 잠그는 게 안전)
    

  

11.3 공공/제도 정보 취급

- 법률 자문처럼 쓰지 않는다. “공식 문서 링크 + 체크리스트”로 제공
    
- 비자/정착 절차는 정부 문서 근거 중심으로 유지(예: 14일 규정은 MOFA/법무성 문서에 명시) 
    

  

12. 기술·아키텍처 제안(유지보수 우선)
    

  

- Front: React 19 + React Router 7 + TailwindCSS 4
    
- Backend: NestJS(모듈/가드/DTO로 규칙 강제)
    
- DB: PostgreSQL(핵심 데이터), Redis(큐/알림), S3 호환 스토리지(문서 파일)
    
- Search: MVP는 DB 필터, 성장 시 Meilisearch/OpenSearch
    
- Auth: 이메일/소셜(OAuth) + RBAC
    
- Observability: 이벤트 로그(문서 리뷰, 모더레이션, 세션 결과)는 반드시 남김
    

  

13. 출시 계획(실행 가능한 순서)
    
    Phase 0(2주): 설계 고정
    

  

- 상태기계 정의(상태/전이/필수 산출물)
    
- 문서 템플릿 1차(일/영 이력서, 職務経歴書, STAR 답변, 오퍼 비교)
    
- 커뮤니티 구조화 스키마 확정(면접후기 필드)
    

  

Phase 1(6~8주): MVP 출시

- 진단/로드맵
    
- 파이프라인 트래커
    
- 문서 센터+리뷰
    
- 멘토 예약(소수 큐레이션)
    
- 커뮤니티(구조화)
    
- 도쿄 정착 체크리스트(최소)
    

  

Phase 2(8~12주): PMF 강화

- 스터디/프로젝트 매칭
    
- 포트폴리오 빌더

Phase 3: 파트너십

- 기업 제휴 → 플랫폼 내 지원
    
- 공공 지원 “옵션 안내” 강화(K-Move/정착지원금 등) 
    

  

14. 리스크 레지스터(숨기면 망함)
    

  

- 멘토링 품질: 자동화 불가. 선발·평가·리포트 강제 없으면 즉시 붕괴
    
- 커뮤니티 품질: 자유글 중심이면 “허언/광고/카더라”로 오염. 구조화+모더레이션이 본체
    
- 법무/규제: 유료 매칭/소개는 위험도가 급상승. MVP에서 선을 명확히 그어야 함
    
- 채용공고 수집: 스크래핑은 실무적으로도, 법적으로도 지속 가능성이 낮음(초기엔 링크 저장 모델)
    
- 정착 정보: 지역별 편차가 커서 “도쿄 우선” 전략이 필요. 정부 문서 기반 체크리스트로 최소화 
    

  

15. 이 기획서에서 “완성” 기준(미작성 방지 체크)
    

  

- 대상/범위/목표/비목표 명시
    
- 5단계 모델을 상태기계로 설계
    
- MVP 기능, 화면(IA), 데이터 스키마, 운영 정책, 출시 로드맵, 리스크까지 포함
    
- 근거가 필요한 제도/통계/기관은 공식 출처 인용(후생노동성, 법무성/출입국, 외무성 등)
